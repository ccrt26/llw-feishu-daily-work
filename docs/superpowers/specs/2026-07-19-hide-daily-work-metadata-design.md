# 工作记录内部数据折叠设计

## 目标

让工作记录中的程序内部编码像“原始内容”一样默认折叠，避免干扰用户阅读，同时保留补充、排序、去重和 U 盘便携能力。

## 最小方案

- 内部数据继续保存在同一个 `工作记录.md`，不迁移存储位置。
- 每条记录前使用 Obsidian 默认折叠 callout：

```markdown
> [!abstract]- 内部数据（程序使用）
> llw-record-data: ENCODED_DATA
```

- `-` 表示默认折叠；用户平时只看到“内部数据（程序使用）”标题，需要时可以展开。
- 原有“原始内容 N｜首次记录/补充”折叠方式保持不变。
- 不再生成 `llw-record-start`、`llw-record-data`、`llw-record-end` HTML 注释。

## 兼容与迁移

- 解析器同时支持现有 HTML 注释格式和新的折叠格式。
- 新建或补充记录统一输出新格式。
- 现有文件在 Mac 本地备份后，通过旧格式解析器读取并使用新渲染器原样重写。
- 迁移前后必须核对记录数量、整理内容、后续事项及全部原文。

## 不变范围

- AI + Skill 继续只负责语义判断和整理。
- 确定性程序继续负责记录 ID、排序、去重和安全写入。
- 不修改状态格式、飞书权限、运行目录、文件目录或每天一个 Markdown 的规则。
- 不新增数据库、旁路索引或跨文件事务。

## 验收

- 新格式默认折叠，效果与“原始内容”一致。
- 展开后内部编码仍完整可用。
- 旧文件可以读取，新文件可以往返解析。
- 完整测试通过，现有工作记录迁移后内容不丢失。
